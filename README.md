# 촬영지 날씨 비교표 앱

기상청은 날짜+지역만 입력하면 자동 조회, 아큐웨더는 직접 입력해서
사진과 동일한 형식의 비교표 이미지를 만드는 도구입니다.

## 배포 방법 (Vercel, 무료)

1. 이 폴더 전체를 GitHub 저장소에 올립니다.
   - GitHub 계정이 없으면 github.com에서 새로 만듭니다.
   - "New repository" → 이 폴더 내용을 업로드 (또는 git push).
2. vercel.com 접속 → GitHub 계정으로 로그인.
3. "Add New... → Project" → 방금 올린 저장소 선택 → Import.
4. **Environment Variables** 항목에 아래 값을 추가:
   - Key: `KMA_SERVICE_KEY`
   - Value: 발급받은 기상청 일반 인증키 (Encoding 안 된 원본 키)
5. "Deploy" 클릭 → 2~3분 후 `프로젝트명.vercel.app` 주소 생성됨.
6. 이후 코드가 수정되면 GitHub에 반영하는 순간 Vercel이 자동 재배포합니다.

## 로컬에서 먼저 테스트하고 싶다면

```bash
npm install
cp .env.example .env.local   # .env.local 파일에 실제 키 입력
npm run dev
```
브라우저에서 http://localhost:3000 접속.

## 알아둘 점 (한계)

- **기상청 10일 이후 날짜는 조회 불가** (기상청이 예보 자체를 제공하지 않음).
- **0~3일 후(단기예보)**: 읍/면/동 단위까지 정밀한 시간별 예보 가능.
- **4~10일 후(중기예보)**: 광역(도) 단위로만 제공되고, 강수확률만 있고 강수량(mm)은 없음.
  `lib/regionCodes.js` 안의 지역코드는 기상청 공식 가이드 기준으로 넣었지만,
  **배포 전에 발급받으신 참고문서(오픈API활용가이드 zip)의 지역코드표와 한 번 대조 확인해주세요.**
  기상청이 코드를 개정하는 경우가 있습니다.
- **주소 → 위경도 변환(지오코딩)**은 무료 서비스(OpenStreetMap Nominatim)를 씁니다.
  - 키 발급이 필요 없다는 장점이 있지만, 초당 요청 제한이 있어서 "전체 기상청 조회"를
    지역이 아주 많을 때 연속으로 누르면 느려지거나 실패할 수 있습니다 (그때는 하나씩 재시도).
  - 아주 작은 단위 주소(예: 특정 리·마을)는 인식 못 할 수 있어요. 그럴 땐 상위 행정구역
    (예: "○○면")까지만 입력해보세요.
- **아큐웨더 값은 자동 조회가 아니라 항상 직접 입력**입니다 (공식 무료 API가 없어서 확정한 방식).
- 표 레이아웃은 지역이 늘어나도 각 표의 크기가 줄어들지 않도록 고정 폭(110px)으로 만들었고,
  한 줄에 2개씩 배치 후 다음 줄로 줄바꿈됩니다.
- **시간단위 상세 모드**: 시작시간을 고르면 그 시각부터 +14시간(총 15개 시점)의 기상청 시간별
  강수확률을 자동 조회합니다. 아큐웨더는 같은 15개 시간대에 맞춰 값만 직접 입력합니다.
  단, 이 모드는 0~3일 후(단기예보)에만 가능하고, 4일 이후(중기예보)는 기상청이 시간별
  데이터를 제공하지 않아 지원하지 않습니다.
- 표의 강수확률 칸은 값이 높을수록 진한 파란색으로, 낮을수록 옅은 색으로 자동 표시됩니다
  (기상청·아큐웨더 공통, 간단 모드/상세 모드 모두 적용).
